# Product Design 参考与原型优化记录

## 1. 本次设计 brief

产品：新闻 RSS 订阅、存储与阅读网站。

目标用户：

- 高频阅读新闻、技术文章、博客和行业动态的人。
- 需要管理大量 RSS 源、排查源异常、快速筛选未读内容的人。

设计目标：

- 让产品第一眼像“信息流控制台”，而不是普通博客列表。
- 保持适合长时间使用的阅读舒适度。
- 把 RSS 产品的关键差异点展示出来：抓取、解析模板、去重、异常排查、源健康。
- 当前阶段以可评审、可点击的本地原型为主，后续再拆成 Vue 3 + TypeScript + Naive UI 组件。

## 2. 参考文档

### 2.1 Naive UI

参考来源：[Naive UI 官网](https://www.naiveui.com/)

可借鉴点：

- Vue 3、TypeScript、主题可定制，符合本项目默认前端偏好。
- 后续实现时可以用 `NLayout`、`NLayoutSider`、`NDataTable`、`NDrawer`、`NTag`、`NButton`、`NInput`、`NSelect`、`NSwitch`、`NProgress` 承接当前原型结构。

落地决策：

- 原型中的侧边栏、表格、抽屉、标签、按钮和输入框都按 Naive UI 可实现组件去设计。
- 当前 HTML 原型保留 8px 或更小圆角，方便后续映射到 Naive UI 主题变量。

### 2.2 Material Design 3

参考来源：

- [Material Design 3 Components](https://m3.material.io/components)
- [Material Design 3 Navigation drawer](https://m3.material.io/components/navigation-drawer/overview)

可借鉴点：

- 组件应按用途组织，导航抽屉适合在大屏设备上切换主要视图。
- 列表、标签、卡片和导航状态要有明确的默认、选中、异常状态。

落地决策：

- 左侧主导航保留稳定位置，避免用户在“阅读、订阅源、模板、日志”之间迷路。
- 标签不当按钮滥用，主要用于过滤、状态和命中提示。
- 异常、抓取中、正常等状态同时使用颜色、文字和形状，不只靠颜色传达。

### 2.3 WCAG 2.2

参考来源：[W3C WCAG 2.2](https://www.w3.org/TR/WCAG22/)

重点约束：

- 文本和背景需要满足足够对比度。
- 不能只用颜色表达信息。
- 小屏和放大场景需要考虑 reflow。
- UI 组件、状态和图形对象需要有可辨识的非文本对比度。

落地决策：

- 深色背景上正文使用高亮灰白，次级文本避免过低对比。
- 状态点旁边始终保留文字，例如“正常、解析异常、网络异常”。
- 移动端单独做阅读预览，不直接压缩桌面三栏。
- 交互控件高度维持在 36px 以上，后续真实实现时继续提升到更稳定的 40px。

### 2.4 仪表盘与复杂应用设计

参考来源：

- [NN/g Dashboards: Making Charts and Graphs Easier to Understand](https://www.nngroup.com/articles/dashboards-preattentive/)
- [NN/g 3 Strategies for Managing Visual Complexity](https://www.nngroup.com/videos/managing-visual-complexity/)

可借鉴点：

- 仪表盘应该帮助用户快速判断是否需要行动。
- 复杂应用应通过稳定位置、清晰层级和渐进披露降低负担。

落地决策：

- 首页不做花哨门户，而聚焦“今日新增、未读、活跃源、异常源、实时信息流、源健康、抓取管线”。
- 抓取和解析细节放到模板实验室，不在首页全部展开。
- 阅读页把筛选、列表、正文分成三栏，减少在列表和正文之间来回跳转。

## 3. 参考设计方向

### 3.1 信息流控制台

参考对象：

- 运维监控仪表盘。
- 开发者工具台。
- 新闻聚合类阅读器。

设计取舍：

- 采用深色背景和细边框建立“控制台”感。
- 关键状态用青色、绿色、琥珀和红色区分。
- 避免大面积渐变和装饰图，核心视觉来自数据层级和状态反馈。

### 3.2 沉浸式阅读器

参考对象：

- Readwise Reader、Feedly、Inoreader 一类信息阅读产品的列表 + 阅读模式。

设计取舍：

- 中间列表负责扫读，右侧正文负责深读。
- 阅读工具条提供字号、行宽、收藏、稍后读、打开原文。
- 正文行宽收窄，避免长行阅读疲劳。

### 3.3 RSS 解析实验室

参考对象：

- API 调试器。
- 数据映射工具。
- 日志排查面板。

设计取舍：

- 左侧模板列表，中间字段映射，右侧原始字段与标准化结果。
- 增加命中路径、置信度、异常提示，帮助用户理解模板为什么生效。
- 保存前必须能预览，不让用户盲改 JSON。

## 4. 当前原型审计

### 4.1 优点

- 深色科技感方向明确，适合 RSS 控制台。
- 首页、阅读页、订阅源管理、模板实验室、移动阅读都已覆盖。
- 页面结构未来可以较自然地映射到 Vue + Naive UI。

### 4.2 风险

- 首页的 RSS 抓取管线不够突出，用户不容易理解“抓取、解析、去重、入库”的产品差异。
- 阅读页已有正文，但缺少阅读偏好工具和内容辅助信息。
- 模板实验室有字段映射，但还不够像一个可操作的调试工具。
- 原型缺少点击反馈，评审时不容易感知信息架构。

### 4.3 本次优化项

- 增加原型顶部切换器，用于快速跳转到不同画板。
- 首页补充 RSS 抓取管线和异常处理队列。
- 阅读页增加阅读偏好工具、AI 摘要提示和相关文章。
- 订阅源页增加健康概览和风险队列，强化运维管理属性。
- 解析模板页增加测试源输入、命中置信度、字段命中路径和异常提示。
- 增加少量 JavaScript 交互：画板跳转、选中文章、筛选切换、模板切换、阅读字号切换、预览状态反馈。
