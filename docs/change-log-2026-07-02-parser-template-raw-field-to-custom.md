# 2026-07-02 解析模板原始字段加入自定义字段

## 实现过程

1. 在解析模板页的“原始字段树”每一行增加“加入”按钮。
2. 点击“加入”后，将当前原始字段路径自动写入“自定义字段”编辑区。
3. 根据原始路径自动生成驼峰格式字段名，例如 `foreignMarkup.commentsUrl.value` 会生成接近 `commentsUrlValue` 的自定义字段名。
4. 增加重复路径判断：如果路径已经存在于自定义字段映射中，不重复添加，并提示已存在字段。
5. 调整字段树布局，支持路径、样例值和操作按钮三列展示。

## 验证命令

```bash
pnpm frontend:build
```

## 执行结果

- 前端 TypeScript 编译通过。
- Vite 生产构建通过。
- 构建提示主 JS chunk 超过 500 kB，属于既有体积提示，不影响本次功能。

## 遗留问题

- 当前字段名生成基于路径规则，后续可进一步支持用户在加入前选择字段命名策略。
- 原始字段树当前最多展示 80 行，深层字段较多时仍需要后续增加搜索或筛选。
